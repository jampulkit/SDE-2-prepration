# Mock Interview Practice — DSA

## How to Use

Set a timer. Solve each problem on paper or a blank editor (no autocomplete). Talk out loud as if an interviewer is watching. After each session, review your solutions against the optimal approach.

---

## Session 1: Arrays + Trees (60 min)

| # | Problem | Time | Difficulty | Pattern |
|---|---------|------|-----------|---------|
| 1 | Two Sum (LC 1) | 10 min | Easy | HashMap lookup |
| 2 | Product of Array Except Self (LC 238) | 20 min | Medium | Prefix/suffix |
| 3 | Binary Tree Maximum Path Sum (LC 124) | 30 min | Hard | DFS, global max |

**What to say while solving:**
1. Read problem, clarify edge cases ("Can the array be empty? Can values be negative?")
2. State brute force and its complexity ("Brute force is O(n²), checking all pairs")
3. Explain your optimized approach before coding ("I'll use a HashMap to store complements")
4. Code cleanly, explain as you write
5. Trace through an example
6. State time and space complexity

## Communication Checklist

- [ ] Clarified constraints and edge cases before coding
- [ ] Stated brute force approach and why it's suboptimal
- [ ] Explained optimized approach clearly before writing code
- [ ] Wrote clean code with meaningful variable names
- [ ] Traced through at least one example
- [ ] Stated time and space complexity
- [ ] Discussed edge cases (empty input, single element, duplicates)
- [ ] Stayed within time limit
