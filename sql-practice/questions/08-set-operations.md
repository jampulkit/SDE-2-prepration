# 08 — Set Operations (UNION, INTERSECT, EXCEPT)

### Q1. [Easy] Combine a list of all employee emails and all customer emails into one list (no duplicates).
-- UNION (deduplicates) vs UNION ALL (keeps duplicates).

### Q2. [Medium] Find customers who are also employees (same email in both tables).
-- INTERSECT.

### Q3. [Medium] Find cities where customers live but no employees work (based on department location).
-- EXCEPT.

### Q4. [Medium] Create a combined "activity feed": show all orders and all payments in chronological order, with a type column ('ORDER' or 'PAYMENT').
-- UNION ALL with a literal column for type.
